# Validator Monitor Pro

Validator Monitor Pro is a high-performance, single-page web dashboard designed for real-time monitoring of validator health and reachability across the Polkadot ecosystem.

By consuming data generated by [subp2p-explorer](https://github.com/lexnv/subp2p-explorer), this tool provides network operators and stakeholders with a clear, actionable overview of node connectivity, DHT records, and peer metadata.

## Key Features

### Multi-Network Support

Seamlessly switch between major ecosystem networks to monitor specific validator sets:
- Polkadot & Kusama (Mainnets)
- Paseo & Westend (Testnets)


### Real-Time Health & Reachability

- Live Status: Immediate visual indicators for reachable vs. unreachable validators.

- DHT Tracking: Verify whether a validator has an active record in the Distributed Hash Table.

- Detailed Analytics: View authority keys, Peer IDs, agent versions, and advertised multiaddresses.

### Advanced Navigation

- Search & Filter: Find nodes instantly by identity, public key, Peer ID, or IP address. Filter by online/offline status to isolate issues.

- Flexible Viewing: Toggle between Full View (detailed metrics) and Compact View (high-density monitoring).

- Smart Sorting: Organize the list by health status or alphabetical name.

### Network Insights & Export

- Summary Cards: At-a-glance stats for total nodes, DHT record counts, and address reachability percentages.

- Portability: Export raw JSON reports for any network for offline analysis or custom tooling.

## Architecture

The dashboard operates as a lightweight frontend consumer:

- Probe: subp2p-explorer crawls the network, probing validator nodes for p2p reachability.

- Report: The crawler generates comprehensive JSON reports containing peer metadata and connection results.

- Render: Validator Monitor Pro fetches these JSON files and transforms the raw data into a searchable, interactive interface.


## Setup

1. Clone and build subp2p-explorer

```bash
  git clone https://github.com/lexnv/subp2p-explorer.git
  cd subp2p-explorer
  cargo build --release
```

2. Run authority checks

  Run the checks for each network to generate the JSON data files. Place the output JSON files alongside index.html.

```bash
  TIMEOUT=120

  # Polkadot
  ./target/release/subp2p-explorer-cli authority-check \
    --url wss://rpc.polkadot.io \
    --timeout "$TIMEOUT" --json polkadot.json

  # Kusama
  ./target/release/subp2p-explorer-cli authority-check \
    --url wss://kusama-rpc.polkadot.io \
    --timeout "$TIMEOUT" --json kusama.json

  # Paseo
  ./target/release/subp2p-explorer-cli authority-check \
    --url wss://rpc.ibp.network/paseo \
    --timeout "$TIMEOUT" --json paseo.json

  # Westend
  ./target/release/subp2p-explorer-cli authority-check \
    --url wss://westend-rpc.polkadot.io \
    --timeout "$TIMEOUT" --json westend.json
```

3. Create the network index

  Create a networks.json file listing the generated JSON files:

```json
  ["polkadot.json", "kusama.json", "paseo.json", "westend.json"]
```

4. Serve the dashboard

  Serve the directory with any static file server:

  python3 -m http.server 8080

  Then open http://localhost:8080 in your browser.
